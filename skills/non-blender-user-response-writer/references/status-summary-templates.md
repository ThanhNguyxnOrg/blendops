# Status Summary Templates

## Purpose
Standardize status framing by mode.

## When to use
At final response synthesis step.

## Templates
- Planning-only: Not Run artifact state
- Runtime blocked: blocker-first template
- Runtime executed: evidence-linked template
- Partial success: caveat-forward template

## Good example
Mode-specific summary with explicit evidence boundary.

## Bad example
Single generic status line for all modes.
