# Stakeholder-Friendly Examples

## Purpose
Provide response examples suitable for non-technical stakeholders.

## When to use
For final output formatting.

## Example classes
- concise executive summary
- implementation-ready checklist summary
- blocked-runtime summary

## Good example
Short, clear status + caveats + action.

## Bad example
Long technical dump with no decision guidance.
