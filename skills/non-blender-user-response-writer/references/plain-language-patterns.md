# Plain-Language Patterns

## Purpose
Provide repeatable patterns for non-technical communication.

## When to use
When writing final user-facing responses.

## Patterns
- lead with user goal
- status in plain terms
- caveat and next action blocks

## Good example
"What you asked / what is ready / what remains unverified / what to do next"

## Bad example
Dense technical paragraph with no action.
