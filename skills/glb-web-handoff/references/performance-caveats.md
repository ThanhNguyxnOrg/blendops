# Performance Caveats

## Purpose
Capture web performance caveats without overclaiming.

## When to use
When discussing web-readiness or optimization expectations.

## Caveat categories
- file size constraints
- material/texture complexity
- runtime test coverage gaps

## Good example
Caveats include user impact and next action.

## Bad example
"Performance is fine" with no basis.
