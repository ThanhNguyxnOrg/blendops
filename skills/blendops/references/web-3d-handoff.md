# Reference: Web 3D Handoff

Status: Draft v0 reference

## GLB expectations

- handoff output should include GLB status, caveats, and next checks
- do not claim GLB exists unless artifact evidence exists

## Transform / scale / origin

- explicitly document assumptions
- if unknown, state as unverified

## Material / texture caveats

- note compatibility caveats for downstream web runtime
- distinguish assumptions vs verified behavior

## Three.js / React Three Fiber handoff notes

- provide compatibility assumptions in plain language
- avoid framework code in this reference

## Artifact status language

Use one of:
- Produced
- Not Produced
- Not Run

Always pair status with caveat and next step.
