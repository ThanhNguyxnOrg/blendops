# evidence-before-done (law summary)

Block readiness/success claims without evidence.

Required behavior:
- include artifact status (Produced/Not Produced/Not Run),
- include verification outcomes,
- include caveats for unresolved checks,
- do not claim preview/render/GLB exists without evidence,
- do not mark Ready if evidence is missing.

Good: "Status: Conditionally Ready. Plan complete; runtime artifacts Not Run."

Bad: "Everything exported and ready." (no evidence)
