# non-blender-user-language (law summary)

Keep outputs clear for users who don't know Blender internals.

Required behavior:
- use plain language for final outputs,
- explain Blender-specific terms when necessary,
- avoid assuming Blender expertise,
- state artifact/runtime status truthfully and clearly.

Good: "The scene plan is ready. Runtime setup is needed before rendering."

Bad: "Eevee viewport shading with HDRI and principled BSDF nodes configured." (too technical without context)
