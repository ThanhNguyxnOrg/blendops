# Laws (Draft v0)

BlendOps laws are non-negotiable constraints and quality gates.

They apply across all BlendOps skills, workflows, recipes, and packs.

Laws are designed to prevent:
- runtime ownership creep
- arbitrary-code user interfaces
- evidence-free completion claims
- Blender-jargon-heavy user-facing output
